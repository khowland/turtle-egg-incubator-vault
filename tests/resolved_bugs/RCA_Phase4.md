# RCA: Phase 4 Test Failures (April 2026)

## 📍 Knowledge Base Location: `tests/resolved_bugs/RCA_Phase4.md`

### 1. StopIteration in Complex Workflows
- **Symptoms**: `next()` call fails to find the SAVE button in `vault_views/3_Observations.py`.
- **Root Cause**: Brittle selectors relying on `type="primary"` or specific `help` text that changed during UI hardening. Also, local `SupabaseMock` classes lacked recursive chain support, causing query returns to be empty/mismatched.
- **Resolution**: Standardized on `b.label == "SAVE"` with `not (b.help)` exclusions for the property matrix. Replaced local mocks with the shared `mock_utils.build_table_aware_mock` supporting deep chains.

### 2. Session Recovery Failures (RES-1 / B-004)
- **Symptoms**: `at.session_state.session_id` mismatched after recovery attempt.
- **Root Cause**: Two-fold:
    1.  Timezone mismatch between `now().astimezone()` and UTC strings.
    2.  **CRITICAL (Agent Zero ID: B-004)**: `supabase_client` was undefined in the `show_splash_screen` function scope, causing silent failure of the recovery query and fallback to a new UUID.
- **Resolution**: Standardized on `now(timezone.utc)` and correctly initialized `supabase_client = get_supabase()` inside the recovery scope.

### 3. Non-Viable Flow De-sync
- **Symptoms**: `AssertionError: Expected status='Dead', got Active`.
- **Root Cause**: A race condition where the mock data was cleared (simulating removal from grid) *before* the update call was verified, or before the app finished its logic loop.
- **Resolution**: Adjusted test timing to ensure the `egg.update` is verified *before* mocking the empty result for the subsequent grid fetch.

### 4. Handoff & State Synchronization (B-008 / B-009)
- **Symptoms**: `AtributeError`, `KeyError`, and `StreamlitAPIException` during view transitions.
- **Root Cause**:
    - **B-008**: Format mismatch between raw ID setter (`2_New_Intake.py`) and split-parsed getter (`3_Observations.py`).
    - **B-009**: Multiselect desync where a newly created bin ID is in session state but missing from the DB-backed multiselect options list.
- **Resolution**: Porting the "Deep Fix" strategy: standardizing on raw ID handoffs and filtering multiselect defaults.

### 5. Initialization Race Conditions (B-015)
- **Symptoms**: `KeyError: None` on first load of Intake.
- **Root Cause**: Accessing the species map before the selectbox has finished initializing.
- **Resolution**: Added null guards and `.get()` safety patterns.
