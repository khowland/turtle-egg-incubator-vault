## 🚩 Identified Issues & Logic Flaws

### 1. Clinical Gap: Missing Intake Mass
- **Description:** Requirement §2 mandates weight recording. The current workflow allows creating bins without weights in 2_New_Intake.py, deferring it to 3_Observations.py.
- **Potential Impact:** Incomplete data if a bin is never checked in the same session.
- **Recommended Fix:** Add a mandatory "Initial Bin Mass (g)" field to the Bin Setup step in Intake.

### 2. Silent Failures in Session Logic
- **Description:** Multiple pass statements in utils/session.py's try...except blocks.
- **Potential Impact:** Biologists may believe they are in a persisted session when the DB connection actually failed.
- **Recommended Fix:** Replace pass with user notifications (e.g., st.warning) where database writes are critical.

### 3. Weak Finder Validation
- **Description:** Finder name regex warning doesn't block the SAVE button in all paths.
- **Potential Impact:** Special characters could break legacy downstream parsers.
- **Recommended Fix:** Ensure is_valid_finder state prevents the RPC call.

### 4. Technical Debt: Header Use Cases
- **Description:** Module headers contain [Pending] sections for practical usage descriptions.
- **Recommended Fix:** Update headers with functional use cases for better maintainability.
