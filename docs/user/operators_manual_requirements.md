# Requirements for the WINC Sovereign Operator Manual Overhaul

## 1. OBJECTIVE
Produce a high-fidelity, comprehensive operator manual that empowers volunteer clinicians to manage turtle subjects with 100% accuracy. The manual must follow the **"Professional-Simplicity"** mandate: Enterprise-grade structure combined with 5th-grade level clarity and extensive visual logic.

---

## 2. AESTHETIC & STRUCTURAL STANDARDS

### A. Cover Page (Institutional Excellence)
- **Requirement**: Use the asset **operators_manual_cover_page.png** for the high-impact, full-page cover unless instructed otherwise. This asset shows a clinical incubator, mother turtle, and hatchlings, and includes WINC identity and Version v10.5.1 labels.

### B. Navigation & Glossary
- **Clickable TOC**: Must allow users to find any task in under 5 seconds.
- **Controlled Vocabulary**: Define a list of "System Terms" (e.g., *Subject*, *Observation*, *Vault*) to be used exclusively. Avoid fuzzy language like "unit" or "item."
- **Glossary of Icons**: Visual guide mapping every stage icon (S0-S6) to its physical real-world equivalent.

### C. 5th-Grade Grammar & Global Clarity
- **Rule**: Short sentences (max 15 words). Active verbs. No jargon without an immediate simple definition.
- **Micro-Copy Alignment**: Every instruction must use the **EXACT** text found on the screen buttons (e.g., if the button says "Establishing Case," the manual cannot say "Click Save").

## 3. THE CLINICAL PROTOCOL (BIOLOGICAL ACCURACY)

### A. The "Air Sac" Mandate (Visual Logic)
- **Requirement**: A cross-section diagram of a turtle egg.
- **Goal**: Explain the *biological reason* why "NEVER ROTATE" is the Golden Rule (oxygen exchange and embryonic drowning). 

### B. Bio-Critical Action Matrix
- **Requirement**: A high-visibility "Red Alert" table.
- **Logic**: Defines immediate actions based on health scores.
    - *Example*: "If Molding or Leaking is Level 3 -> DO NOT SAVE -> Call Senior Biologist immediately."

### C. Environmental Protocol
- **Requirement**: Guidance for handling subjects in 30°C+ heat or direct sunlight.
- **Goal**: Minimize exposure time during the weighing and property assessment process.

## 4. GOVERNANCE & DATA INTEGRITY (RED TEAM STANDARDS)

### A. Surgical Correction Policy
- **Rule**: No blanks allowed.
- **Requirement**: Instructions must specify that a "Void Reason" is mandatory for every correction. Explain that these are permanent audit records.

### B. Crisis Mode Transcription (Double-Witness)
- **Rule**: In connectivity failure scenarios, paper logs must be transcribed back into the vault.
- **Requirement**: Include a "Second Signature" requirement for transcription to prevent SUBJECT ID typos.

### C. Error Message Lookup Table
- **Requirement**: A "Master Error Index."
- **Logic**: Map technical errors (e.g., "Postgres 42P01") to user-level solutions (e.g., "Check internet" or "Re-login").
