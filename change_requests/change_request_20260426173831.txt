TITLE
-----
Mobile Sidebar: Restore Collapse/Uncollapse Toggle Button

DESCRIPTION
-----------
On mobile devices, the sidebar navigation menu has no visible way to
collapse or expand it after it has been collapsed. The hamburger/toggle
button that Streamlit renders into the top header bar is being
completely suppressed by a CSS rule in bootstrap.py that was added
during the Tight-Fit mobile layout work (CR-20260423 / Bug-PERF-001).

ROOT CAUSE
----------
In utils/bootstrap.py, the global CSS block contains:

    [data-testid="stHeader"] {
        height: 0rem !important;
        visibility: hidden !important;
        min-height: 0rem !important;
    }

This rule kills the entire Streamlit top header bar, which on mobile is
the ONLY container that holds the sidebar collapse/expand button
(the hamburger icon). On desktop, this is acceptable because screen
width keeps the sidebar always visible. On mobile, once the sidebar
collapses (auto-collapses on page load or navigation), there is no
recovery affordance for the user. The clinical team cannot access the
navigation menu.

NOTE: The stHeader suppression was intentional for the Tight-Fit
desktop layout and Bug-PERF-001 fix. The fix must NOT revert that work
for desktop. Only restore the header for mobile breakpoints.

PROPOSED FIX
------------
Replace the flat [data-testid="stHeader"] hide rule with a
responsive CSS rule that:
  - Desktop (>= 768px): keeps height=0 / visibility=hidden (no change)
  - Mobile (< 768px):   restores height to auto and visibility=visible
    so the hamburger toggle is reachable.

Single change: utils/bootstrap.py — replace the stHeader block with a
@media-query-gated version. No Python logic changes required.
