Step 2 section
for new intake under bin configuration
Total eggs was 1 (should be 0 for a new bin).
For a new bin there should be 0 Total Eggs intially. And we only need one column "Add Eggs" for user to enter new egg count to new bin. 

For new intake and new bin, we don't need the following columns for Bin Configuration data entry.
remove these columnms from the data entry grid and make sure they are nullable. 
- Initial Mass (g)
- Incubator Temp (F)

"Initial Mass (g)" and "Incubator Temp (F)" should be entered as part of Bin Observations on next screen.

When SUPPLEMENTAL INTAKE radio button selected
radio button "Supplemental Intake" should be labelled "Existing Intake: Add new bins or eggs"
If a user selects radio button "Supplemwental Intake", then user is presented with dropdown of existin intakes, and they choose one.
The bin configuration for that intake will load and show the existing bin(s) and egg counts for that intake. The existing egg counts should be read-only. 
Now Bin configuraiton grid will have the "new_egg_count" (change column label to Add New Eggs) data entry column defaulted to 0, allowing additional eggs to be added.

Summary
We require a read only column in bin configuration grid to show "Current Egg Count", and a data entry column "Add New Eggs" .
For new intake and new bin: "Current Egg Count" will be 0, and they will enter new egg count under column "Add New Eggs"
For existing intake, and existing bin: "Current Egg Count" will show egg count from DB, and they will enter new egg count under column "Add New Eggs"
For existing intake and new bin: "Current Egg Count" will be 0, and they will enter the new egg count under column "Add New Eggs"
